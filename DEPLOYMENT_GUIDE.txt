SECTION 1 - DEPLOYMENT OPTIONS (FREE)
================================================================

RECOMMENDED: Streamlit Community Cloud
  URL: https://share.streamlit.io
  Cost: Free forever
  Limits: 1 GB RAM, 1 CPU, public repos
  Best for: Streamlit apps - purpose-built, zero config
  Time to deploy: ~5 minutes

ALTERNATIVE: Render
  URL: https://render.com
  Cost: Free tier (spins down after 15 min inactivity)
  Best for: When you need FastAPI as a separate service
  Time to deploy: ~10 minutes

NOT RECOMMENDED: Vercel / Netlify
  Reason: These are for static sites and serverless functions.
  UIP needs a persistent Python process. They will NOT work.

================================================================
SECTION 2 - DEPLOY ON STREAMLIT COMMUNITY CLOUD (RECOMMENDED)
================================================================

HOW IT WORKS:
  Streamlit Cloud runs app.py directly.
  The _start_backend() function in app.py automatically launches
  the FastAPI server as a subprocess on port 8000 when the app loads.
  Both services run on the same container - no extra config needed.

STEP 1 - PREPARE YOUR GITHUB REPOSITORY

  Make sure these files are in your repo root:
    app.py                  (Streamlit entry point)
    requirements.txt        (Python dependencies)
    packages.txt            (System packages - libgeos-dev)
    .streamlit/config.toml  (Streamlit config)
    src/                    (All Python modules)
    pages/                  (All 11 pages)
    data/                   (CSV data files)

  Files NOT needed in repo (already in .gitignore):
    .env                    (use Streamlit Secrets instead)
    __pycache__/
    marketing_sample_*.csv  (large raw Kaggle file - 30MB)
    test_*.py

STEP 2 - PUSH TO GITHUB

  If starting fresh:
    git init
    git add .
    git commit -m "Initial deployment commit"
    (Create new repo on github.com, then:)
    git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
    git branch -M main
    git push -u origin main

  If repo already exists:
    git add .
    git commit -m "Deployment ready"
    git push

STEP 3 - DEPLOY ON STREAMLIT CLOUD

  1. Go to https://share.streamlit.io
  2. Sign in with your GitHub account
  3. Click "New app" (top right)
  4. Fill in:
       Repository: YOUR_USERNAME/YOUR_REPO
       Branch: main
       Main file path: app.py
  5. Click "Deploy!"
  6. Wait 3-5 minutes for the build to complete

STEP 4 - ADD YOUR GROQ API KEY (for AI Insights)

  In Streamlit Cloud dashboard:
  1. Click your app name
  2. Click "Settings" (top right)
  3. Click "Secrets" in the left menu
  4. Add this exactly:

       GROQ_API_KEY = "gsk_your_key_here"

  5. Click "Save" - app restarts automatically

  Get a free Groq key at: https://console.groq.com
  No credit card required. 14,400 requests/day free.

STEP 5 - VERIFY DEPLOYMENT

  Your app URL will be:
    https://YOUR_USERNAME-YOUR_REPO-app-HASH.streamlit.app

  Check these work:
    - Overview page loads with KPI cards
    - Simulator runs a scenario
    - Geo Career Advisor shows the India map
    - AI Insights shows LLM response (if key added)

SECTION 3 - DEPLOY ON RENDER (ALTERNATIVE)
================================================================

Render runs FastAPI and Streamlit as separate services.
Use this if you want the API independently accessible.

STEP 1 - PUSH TO GITHUB (same as Section 2, Step 2)

STEP 2 - CREATE RENDER ACCOUNT
  Go to https://render.com and sign up with GitHub

STEP 3 - DEPLOY THE API BACKEND FIRST

  1. Click "New +" then "Web Service"
  2. Connect your GitHub repo
  3. Fill in:
       Name: uip-api
       Runtime: Python 3
       Build Command: pip install -r requirements.txt
       Start Command: uvicorn src.api:app --host 0.0.0.0 --port $PORT --log-level error
  4. Under "Environment Variables" add:
       GROQ_API_KEY = your_key_here
  5. Click "Create Web Service"
  6. Wait for deployment. Note your API URL:
       https://uip-api.onrender.com

STEP 4 - DEPLOY THE FRONTEND

  1. Click "New +" then "Web Service"
  2. Connect the same GitHub repo
  3. Fill in:
       Name: uip-frontend
       Runtime: Python 3
       Build Command: pip install -r requirements.txt
       Start Command: streamlit run app.py --server.port $PORT --server.address 0.0.0.0 --server.headless true
  4. Under "Environment Variables" add:
       API_BASE_URL = https://uip-api.onrender.com
       GROQ_API_KEY = your_key_here
  5. Click "Create Web Service"

  Your frontend URL: https://uip-frontend.onrender.com

  NOTE: Free Render services spin down after 15 min of inactivity.
        First load after inactivity takes ~30 seconds. This is normal.

SECTION 4 - ENVIRONMENT VARIABLES REFERENCE
================================================================

GROQ_API_KEY
  Required: No (optional)
  Where: Streamlit Secrets or Render Env Vars
  Purpose: Groq LLaMA 3.1 for AI Insights
  Get free at: https://console.groq.com

GEMINI_API_KEY
  Required: No (optional)
  Where: Same as above
  Purpose: Google Gemini fallback for AI Insights
  Get free at: https://aistudio.google.com/app/apikey

OPENAI_API_KEY
  Required: No (optional, paid)
  Where: Same as above
  Purpose: OpenAI GPT fallback for AI Insights

API_BASE_URL
  Required: Only on Render (frontend service)
  Where: Render Env Vars on the frontend service
  Default: http://127.0.0.1:8000
  Set to: https://uip-api.onrender.com

NOTE: Without any API keys, the app works fully.
      AI Insights uses rule-based fallback instead of LLM.
      All other features work without any API keys.

================================================================
SECTION 5 - WHAT HAPPENS ON FIRST LOAD
================================================================

  1. Streamlit loads app.py
  2. _start_backend() checks if FastAPI is running on port 8000
  3. If not running, starts: uvicorn src.api:app --port 8000
  4. Waits up to 8 seconds for backend to be ready
  5. Overview page loads and calls /data-status
  6. World Bank API queried for live India unemployment data
  7. If World Bank API unavailable, falls back to local CSV
  8. KPI cards, forecast chart, recession risk score render
  9. User can navigate to any of the 11 pages

  First-load time: 10-20 seconds (World Bank API call)
  Subsequent pages: 2-5 seconds (data cached 24 hours)

SECTION 6 - TROUBLESHOOTING
================================================================

ISSUE: App shows "API Offline" warning
  FIX: Check Streamlit Cloud logs for Python errors
       Most common cause: missing package in requirements.txt

ISSUE: ModuleNotFoundError: No module named 'folium'
  FIX: Ensure requirements.txt has: folium>=0.15.0
       Ensure packages.txt has: libgeos-dev

ISSUE: Map not rendering on Geo Career Advisor
  FIX: Ensure streamlit-folium is in requirements.txt

ISSUE: World Bank data shows "Offline - Local CSV"
  FIX: Temporary API issue. App auto-falls back. No action needed.

ISSUE: AI Insights shows "Rule-based fallback"
  FIX: Add GROQ_API_KEY in Streamlit Cloud Settings > Secrets

ISSUE: Git push rejected (file too large)
  FIX: The marketing_sample_*.csv is in .gitignore already.
       Only job_postings_sample.csv (7MB) is committed - fine.

ISSUE: Render app slow on first load
  FIX: Normal on free tier. Service wakes up in ~30 seconds.
